# Cryptool 仕様書

## 概要
AES-256 および PGP 共通鍵暗号化されたメモファイルを安全に閲覧、および新規作成するための Web 拡張機能。

## 機能要件
- **対応フォーマット**:
    - AES-256 GCM (PBKDF2 鍵派生)
    - OpenPGP Symmetric Encryption (AES-128/192/256, SHA-1/256/384/512 S2K 対応)
- **復号/暗号化プロセス**:
    - ブラウザ内（ローカル）ですべての処理を行う。
    - 外部サーバーへのデータアップロードは一切行わない。
- **UI/UX**:
    - ダークモードに対応したモダンなグラスモーフィズムデザイン。
    - 拡張機能アイコンクリック時のメニューから、独立したタブ（Viewer）を起動。
    - 復号鍵（パスワード）の表示/非表示切り替え。
    - メモ閲覧時のフォントサイズ調整（スライダーによるリアルタイム変更）。
    - 揮発性保護（Auto-clear on hide）: タブが非表示になった際にメモリと表示を自動クリア（ON/OFF設定可能）。
    - **新規作成**: 平文テキストを入力し、AES-256 GCM (v1) 形式で暗号化保存（ダウンロード）する機能。
- **制約事項**:
    - 既存の暗号化ファイルの「編集・上書き保存」機能は持たない（閲覧または新規作成のみ）。
    - 範囲選択およびコピーはサポートする。
    - 拡張機能内にパスワードや平文などの情報を一切保存しない。

## 技術スタック
- **プラットフォーム**: Web Extension (Manifest V3)
- **ブラウザ互換性**: Firefox (Gecko), Chrome (Chromium)
- **暗号化ライブラリ**:
    - Web Crypto API (AES-GCM)
    - OpenPGP.js (PGP Support)

## AES-GCM ファイルフォーマット

### v1 フォーマット（現行）

| フィールド | サイズ | 内容 |
|---|---|---|
| Magic | 4 bytes | `CRTL` (0x43 0x52 0x54 0x4C) |
| Version | 1 byte | `0x01` |
| Iterations | 4 bytes | PBKDF2 反復回数（big-endian uint32）|
| Salt | 16 bytes | PBKDF2 ソルト（乱数）|
| IV | 12 bytes | AES-GCM 初期化ベクタ（乱数）|
| Ciphertext + Tag | 可変 | 暗号文 + GCM 認証タグ(16 bytes) |

- 鍵派生: PBKDF2-SHA-256、推奨反復回数 600,000
- 暗号化: AES-256-GCM

### レガシーフォーマット（v0）

| フィールド | サイズ | 内容 |
|---|---|---|
| Salt | 16 bytes | PBKDF2 ソルト |
| IV | 12 bytes | AES-GCM 初期化ベクタ |
| Ciphertext + Tag | 可変 | 暗号文 + GCM 認証タグ(16 bytes) |

- 反復回数は 100,000 固定（フォーマット外で固定値）
- v1 フォーマットとの判別: 先頭4バイトが `CRTL` でなければレガシーとみなす

### フォーマット判別ロジック

```
先頭4バイト == 'CRTL' かつ バイト長 >= 37
  → v1: ヘッダーから iterations を読む
それ以外
  → レガシー: iterations = 100000 固定
```
